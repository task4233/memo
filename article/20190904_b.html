          # B - Kleene Inversion

          # 概要

          • 長さNNNの整数列AAAをKKK回繰り返した整数列をBBBとする
          • BBBの転倒数を109+710^9 + 7109+7で割ったあまりを求めよ

          # 思考

          • 手元で実験すると, AAAの要素ごとの転倒数をCiC_iCi​とすると, ∑i=0n−1Ci∗k∗(k+1)/2\sum_{i=0}^{n-1} {C_i * k * (k+1) / 2}∑i=0n−1​Ci​∗k∗(k+1)/2で求められそう?
          • 実装する
          • テストケース3で詰まる
          • MODの取り方がよくない?
          • 直しても通らない

          ここで30分終了

          • editorialを見る
          • AAAの内部のみで完結している転倒数と, AAAの内部およびAAAの外部とでできる転倒数の2つに分けて考えると良いとのこと
            • 元々の実装だと, Aが単調増加の時のテストケースに対応していなかった

          # 解法

          • AAAの内部のみで完結している転倒数は, O(N2)O(N^2)O(N2)で求められる
            • Ai,Aj(i<j)A_i, A_j(i < j)Ai​,Aj​(i<j)を比較して, Ai>AjA_i > A_jAi​>Aj​となっている数をカウント
            • AAAの内部のみで完結している転倒数はKKK個あるので, KKK倍する
          • AAAの内部およびAAAの外部とでできる転倒数も, O(N2)O(N^2)O(N2)で求められる
            • Ai,Aj(i<j)A_i, A_j(i < j)Ai​,Aj​(i<j)を比較して, Ai>AjA_i > A_jAi​>Aj​となっている数をカウント
            • AAAの内部およびAAAの外部とでできる転倒数は, AiA_iAi​とAjA_jAj​を選ぶので, K×(K−1)/2K\times(K-1)/2K×(K−1)/2倍する
          • これらの和を取れば良い
